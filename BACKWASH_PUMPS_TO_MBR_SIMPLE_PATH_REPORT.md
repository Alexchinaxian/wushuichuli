## 修改需求
两个反洗泵到MBR膜池的连接线都应该是：
1. 从反洗泵的左边点先水平移动出来
2. 然后直接垂直向下到MBR膜池的连接点
3. 每个反洗泵独立连接，不需要汇聚

## 修改内容

### 1. 原始设计分析
**复杂路径**（修改前）:
```
1#反洗泵: (900,230) → (880,230) → (880,270) → (880,420) → (875,420)
2#反洗泵: (900,310) → (880,310) → (880,270) → (880,420) → (875,420)
```
- 问题: 两个反洗泵在中间点(880,270)汇聚
- 问题: 路径复杂，有额外的垂直移动
- 不符合"直接竖线到连接点"的要求

### 2. 简单路径设计
**简单路径**（修改后）:
```
1#反洗泵: (900,230) → 水平向左到(880,230) → 垂直向下到(880,420) → 水平向左到MBR膜池右边点(875,420)
2#反洗泵: (900,310) → 水平向左到(880,310) → 垂直向下到(880,420) → 水平向左到MBR膜池右边点(875,420)
```
- 特点: 每个反洗泵独立连接
- 特点: 先横再竖，路径简洁
- 符合"从左边点横先出来直接竖线到连接点"的要求

### 3. 代码修改
```csharp
// 修改前（复杂汇聚路径）
double 反洗泵水平X = 880;
double 反洗泵垂直起点Y = 270;
AddOrtho(反洗泵1_左, new Point(反洗泵水平X, 230), FlowLineType.Backwash);
AddOrtho(new Point(反洗泵水平X, 230), new Point(反洗泵水平X, 反洗泵垂直起点Y), FlowLineType.Backwash);
AddOrtho3(new Point(反洗泵水平X, 反洗泵垂直起点Y), new Point(反洗泵水平X, 420), MBR膜池_右边点, FlowLineType.Backwash);

// 修改后（简单独立路径）
AddOrtho(反洗泵1_左, new Point(880, 230), FlowLineType.Backwash);
AddOrtho3(new Point(880, 230), new Point(880, 420), MBR膜池_右边点, FlowLineType.Backwash);
```

## 设计原则

### 1. 连线模式
- **先横**: 从反洗泵左边点水平向左移动20像素
- **再竖**: 直接垂直向下到Y=420
- **最后横**: 水平向左5像素到MBR连接点

### 2. 独立连接
- **1#反洗泵**: 独立路径，不与其他反洗泵汇聚
- **2#反洗泵**: 独立路径，不与其他反洗泵汇聚
- **简洁明了**: 每个反洗泵有自己的清晰路径

### 3. 正交连线
- **全部正交**: 所有连线都是水平或垂直
- **无斜线**: 使用`AddOrtho3`确保正交路径
- **整齐规范**: 符合工业控制系统连线标准
