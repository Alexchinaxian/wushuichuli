# 工业控制上位机开发计划

## 1. 项目概述

### 1.1 目标
基于现有 WPF 工程，完成一套面向 **MBR 污水处理工艺** 的工业控制上位机（HMI），实现工艺流程可视化、PLC 通信、报警管理、参数设置与历史数据查看，并支持单机部署。

### 1.2 技术栈（当前）
| 类别     | 技术 |
|----------|------|
| 框架     | .NET 8.0, WPF |
| 架构     | MVVM (CommunityToolkit.Mvvm), 依赖注入 (Microsoft.Extensions.DependencyInjection) |
| 通信     | Modbus TCP (NModbus) |
| 数据     | SQLite (Microsoft.EntityFrameworkCore.Sqlite), 内存库当前为 `:memory:` |
| 图表     | LiveCharts.Wpf |
| 流程图   | WPF 原生 Canvas + 配置驱动 (CoordinatesConfig.json) |

### 1.3 参考依据
- **工艺流程与设备布局**: `plans/drawio_设备布局分析.md`（32 设备、27 条连接线）
- **PLC 点位表**: `参考资料/中信国际污水项目PLC点位.md`（西门子 S7 Smart200：I/Q/M/V 区）
- **坐标配置**: `IndustrialControlHMI/Config/CoordinatesConfig_README.md`

## 2. 工程现状扫描

### 2.1 已实现模块
| 模块         | 说明 |
|--------------|------|
| 主框架       | MainWindow 三栏布局（导航 220px / 主内容 / 右侧 300px），标题栏、状态栏、PLC 连接状态 |
| 工艺流程图   | FlowchartView + FlowchartRenderer，ProcessUnitControl / FlowLineControl，配置驱动布局与连接线 |
| 报警管理     | AlarmManagementView、AlarmRepository、AlarmRecord 模型，SQLite 存储 |
| 参数设置     | SettingsView、SettingsManager、SettingRepository，Modbus IP/Port/SlaveId/轮询与超时 |
| 通信抽象     | IModbusService（连接、读/写寄存器、事件）、DefaultModbusConfig |
| PLC 绑定     | PlcDataBindingService，订阅 Modbus 事件、向流程图/报警推送数据 |
| 点位映射模型 | PlcPointMapping、PlcPointMappingProvider（与流程图单元/设备关联） |

### 2.2 待完善与缺口
| 项目         | 现状 | 说明 |
|--------------|------|------|
| Modbus 真实通信 | ModbusService 可能为模拟/占位 | 需接真实 TCP + 寄存器读写，与 NModbus 对接 |
| PLC 协议对齐   | 参考为西门子 I/Q/M/V | 若现场为 Modbus 网关或另一 PLC，需明确寄存器表与地址映射 |
| 流程图与 PLC 联动 | 有 PlcPointMapping，绑定是否完整未全验证 | 需按点位表完成 故障/运行/液位 等与界面元素一一对应 |
| 历史数据页     | 导航有入口，CurrentViewModel 为空 | 需独立 HistoryView + 历史曲线/表格，数据源与存储设计 |
| 数据库持久化   | 使用 `:memory:` | 改为文件 SQLite（如 alarms.db），并设计历史数据表 |
| 报警与 PLC 联动 | 报警有模型与 UI | 需从 PLC 故障点（如 I 区）驱动报警产生/确认/清除 |
| 参数写入       | 设置页仅保存本机配置 | 需支持将设定值、手动/自动等写入 PLC（若协议支持） |
| 打包部署       | 未配置 | 安装包、.NET 运行时/单文件、WebView2 已移除，无额外运行时依赖 |

## 3. 开发计划（分阶段）

### 第一阶段：通信与数据基础（优先级：高）

| 序号 | 任务 | 产出 | 依赖 |
|------|------|------|------|
| 1.1 | 明确现场 PLC 类型与寄存器表 | 文档：寄存器表（Modbus 或 S7 地址映射到 Modbus） | 现场/项目确认 |
| 1.2 | ModbusService 真实 TCP 实现 | 使用 NModbus 的 TcpClient，实现 Connect/Disconnect、ReadHoldingRegisters/ReadInputRegisters、WriteSingleRegister 等 | 无 |
| 1.3 | 从配置或代码读取连接参数 | 启动时从 SettingsManager 读 Modbus IP/Port/SlaveId，连接失败重试与状态回调 | 1.2 |
| 1.4 | PLC 点位与界面映射表固化 | 基于《中信国际污水项目PLC点位》与流程图 ElementId，完善 PlcPointMappingProvider 或 JSON 配置，包含 I/Q/M/V 与 Modbus 地址对应关系 | 1.1 |
| 1.5 | PlcDataBindingService 按地址轮询 | 按映射表定时读 PLC，推送 DataReceived/报警状态到 FlowchartViewModel、报警服务 | 1.2、1.4 |

**验收标准**：上位机可连接真实或仿真 PLC，流程图单元状态（运行/故障/停止）随 PLC 数据变化。

---

### 第二阶段：报警与参数联动（优先级：高）

| 序号 | 任务 | 产出 | 依赖 |
|------|------|------|------|
| 2.1 | 报警与 PLC 故障点绑定 | 从 I 区（或映射后的 Modbus 线圈/寄存器）驱动报警生成，写入 AlarmRecords | 1.5 |
| 2.2 | 报警确认/清除 | 界面确认、清除操作写回数据库；若 PLC 支持，可写 M 区或寄存器 | 2.1 |
| 2.3 | 参数设置页与 PLC 写入 | 液位设定、延时、启停时间等可写项写入 V 区（或对应 Modbus 寄存器） | 1.2、1.4 |
| 2.4 | 关键参数显示与 PLC 同步 | 主界面右侧关键参数（液位、状态等）来自 PlcDataBindingService 或 Modbus 轮询 | 1.5 |

**验收标准**：故障发生时自动记报警并可确认/清除；关键参数与设定值可从界面写入并生效。

## 4. 风险与假设

| 风险 | 缓解 |
|------|------|
| 参考点位表为西门子 S7，现场为 Modbus | 事先确认现场协议；若为 S7，需 Modbus 网关或后续增加 S7 驱动/OPC |
| 寄存器地址与文档不一致 | 以现场为准，维护一份“现场寄存器表”并版本化 |
| 历史数据量过大 | 做采样间隔、归档策略（如只保留 30 天明细）与分页查询 |

**假设**：现场 PLC 支持 Modbus TCP 或通过网关转 Modbus；Windows 10/11 运行环境；单机部署为主。
